# Resume Content Optimization Design

**Goal**

将当前作品集文案从“信息罗列型”调整为“岗位匹配型”，主线聚焦在复杂业务前端、性能优化、工程效率与 AI 提效补充能力。

**Scope**

- 保持现有页面结构与视觉样式不变
- 仅调整 [src/App.jsx](C:\Users\Jmr03\Desktop\fe-projects\minrui-frontend-portfolio\src\App.jsx) 中的文案数据
- 优化 summary、指标区、核心能力、项目经历、时间线与联系区措辞

**Content Direction**

1. 首页 summary 强调复杂业务、多端交付、性能优化和工程化落地。
2. 指标区从泛化标签改为招聘方更容易快速理解的价值标签。
3. 核心能力区减少堆词，改为业务开发、工程化、性能优化、AI 协作四类。
4. 项目经历统一采用“场景 + 动作 + 结果”表达，突出 0 到 1、复杂交互、性能收益和交付效率。
5. AI 能力保留，但改成职业化描述，不使用口语化术语。

**Non-Goals**

- 不新增模块
- 不修改视觉层级与交互行为
- 不新增下载简历、外链或多语言切换
