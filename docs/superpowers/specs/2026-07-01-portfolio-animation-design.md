# 简历网站动画增强设计

## 目标

为当前 React + Vite 简历网站增加更丰富的动画效果，让页面第一眼更像一个强交互作品集，同时保持简历内容清晰、可快速阅读。

本次方向为：

- 视觉感受偏强交互作品感。
- 首屏 Hero 是主要动效舞台。
- 页面保持正常上下滚动，不做锁屏式演示。
- 动画体系统一改为 GSAP，不再使用 framer-motion。

## 当前上下文

项目主体位于 `src/App.jsx` 和 `src/styles.css`。当前页面已经包含：

- 首屏身份介绍和候选人信息卡。
- quick facts 指标区。
- AISDLC workflow tab 交互区。
- evidence、cases、skills、timeline、contact 等内容模块。
- 现有轻量 framer-motion 入场动画。

新增动画应服务于这些现有模块，不改变简历主线、不新增无关功能。

## 非目标

- 不重写页面内容和信息架构。
- 不新增下载简历、多语言、路由或后台能力。
- 不做全屏锁滚动、长时间 pinning 或必须按固定节奏观看的叙事页面。
- 不同时维护 framer-motion 和 GSAP 两套动画体系。

## 方案选择

采用“Hero 强互动，其他区域轻增强”的方案。

对比方案：

- Hero 强互动，其他区域轻增强：首屏有明确记忆点，后续阅读不被打断，复杂度可控。选用此方案。
- Hero + AISDLC 双主舞台：更突出 AI 前端流程，但滚动绑定更重，移动端和调试成本更高。
- 全页面系统化强动效：整页都更炫，但容易喧宾夺主，不适合作为招聘方快速阅读的简历页。

## Hero 动效设计

首屏保持当前布局：左侧是身份、标题、摘要和按钮，右侧是候选人信息卡。

进入页面时按层级播放：

1. eyebrow 轻微淡入。
2. 主标题按词组或行做 stagger 上浮。
3. 摘要和按钮随后进入。
4. 候选人信息卡稍晚出现，形成主次节奏。

背景网格增加轻微流动和鼠标位置响应。效果应克制，避免粒子化或游戏化。

鼠标移动时，候选人信息卡做有限幅度 3D tilt。卡片内部的 monogram、联系项和边框高光有轻微视差，让首屏有可交互感。

按钮 hover 增加高光扫过或边框流动，并保留轻微位移反馈。

首屏滚动离开时，背景层和候选人卡片可以有轻微反向视差，但不 sticky、不锁滚动。

移动端不做鼠标相关交互，只保留入场、按钮反馈和轻量滚动视差。

## 其他区域动效设计

Hero 之后的模块以辅助动效为主，增强浏览节奏，不抢内容。

`quick-facts`：四个指标卡滚动进入时错峰上浮。数值可以做一次轻量揭示或计数效果，不做循环动画。

`AISDLC workflow`：保留当前点击 tab 交互。点击切换时，active 背景、边框高光、右侧标题和正文做短过渡。滚动进入该区域时整体有轻微横向视差，不做 sticky 流程演示。

`evidence` 和 `cases`：列表项滚动进入时按行错峰。案例卡 hover 时做轻量前后景分层，业务问题、我的贡献、结果三列有细微延迟进入。

`skills`：技能标签以小幅 stagger 出现。

`timeline`：每条经历的时间和内容分别进入，形成履历推进感。

`contact`：底部联系区做一次较强但短促的入场，用于页面收尾。

## 技术设计

依赖层面：

- 新增 `gsap`。
- 移除 `framer-motion`。
- 更新 `package.json` 和 `package-lock.json`。

React 层面：

- 将 `App.jsx` 中的 `motion.*` 元素改回普通 HTML/React 元素。
- 使用 `useRef` 标记 Hero、候选人卡片和主要内容模块。
- 使用 `useEffect` 注册 GSAP、ScrollTrigger、鼠标移动事件。
- 使用 `gsap.context()` 限定动画作用域。
- 在 effect cleanup 中清理 timeline、ScrollTrigger 和事件监听，避免热更新后重复绑定。

CSS 层面：

- 为 Hero 增加背景动画层和 3D 透视环境。
- 为候选人卡片、按钮、案例卡片补充 hover 和高光所需样式。
- 只对需要动画的元素添加 `will-change`。
- 保留并加强 `prefers-reduced-motion: reduce` 规则，关闭连续动画、强位移和视差。

## 动效边界

- 桌面端可以有明显交互和滚动视差。
- 移动端减少横向位移、tilt 和鼠标相关效果。
- 所有动画都应有上限：位移小、时间短、缓动统一。
- 不让动画改变内容阅读顺序。
- 不让动画造成文字重叠、横向溢出或布局跳动。

## 验证标准

1. 首屏进入动画、鼠标响应、候选人卡片 tilt、按钮高光都可感知。
2. Hero 是整页最强动效区域，其他模块只增强节奏。
3. 页面仍可正常上下滚动，招聘方能快速阅读简历内容。
4. 移动端没有依赖鼠标的关键效果，没有横向溢出和文字重叠。
5. `prefers-reduced-motion: reduce` 下关闭连续动画、强视差和 tilt。
6. `npm run build` 通过。
7. 如本地浏览器可用，检查桌面和移动端首屏、AISDLC、案例区、底部联系区的视觉状态。
