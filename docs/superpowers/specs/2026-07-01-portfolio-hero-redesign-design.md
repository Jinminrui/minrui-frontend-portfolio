# 作品集首屏重设计规格

## 目标

重新设计当前简历网站首屏，让它从“左右布局的信息展示”变成更有未来感的个人品牌封面。

首屏需要第一眼传达：

- 金敏睿 / Minrui Jin 是页面主角。
- 个人定位是 AI 前端开发工程师。
- 方向与 AISDLC、Agent、MCP、代码审查、知识沉淀等 AI 研发流程相关。
- 页面仍然具有成熟业务前端的可信度，后续内容能自然承接履历、案例和时间线。

本次只设计首屏，不重写整站内容主线，不新增下载简历、多语言、路由或后台能力。

## 当前上下文

项目是 React + Vite 单页作品集，核心文件是 `src/App.jsx` 和 `src/styles.css`。当前版本已经使用 GSAP 和 ScrollTrigger，包含首屏入场、滚动 reveal、Hero 背景流动、鼠标 tilt 和按钮高光。

现有首屏采用左右布局：左侧身份介绍，右侧候选人详细信息卡。用户已明确希望：

- 动画更丰富。
- 首屏可以不展示详细个人信息。
- 不拘泥于现有左右结构。
- 不做工作台、Dashboard 或信息卡堆叠式设计。
- 最终方向为未来感个人品牌。

## 非目标

- 不把首屏做成 AI 工作台、SaaS 控制台或后台系统界面。
- 不在首屏展示电话、邮箱等详细联系方式。
- 不新增无关功能或新的信息架构分支。
- 不引入新的动画库。
- 不做锁屏滚动、长时间 pinning 或必须按固定节奏观看的叙事页面。

## 方案选择

采用“中心封面式 Hero”方案。

对比方案：

- 中心封面式 Hero：无左右栏，姓名和身份成为视觉中心，抽象动态视觉场铺满首屏。它最符合未来感个人品牌，也能避免模板化简历布局。选用此方案。
- 未来感左右构图：左侧文案、右侧抽象视觉。结构清晰，但仍接近现有 Hero，重设计感不足。
- 系统图式 Hero：直接表达 AI 参与研发流程。概念强，但信息密度高，容易滑向工作台或 Dashboard。

## 首屏内容设计

首屏主体由三层组成：

1. 中心标题层。
2. 全屏动态视觉层。
3. 底部事实刻度层。

中心标题层是首屏主阅读路径，包含：

- `金敏睿 / Minrui Jin`
- `AI 前端开发工程师`
- 一句压缩后的摘要，表达“把 AI 接入需求理解、工程实现、Review 和知识沉淀”。
- 保留现有主要行动按钮，但不增加新功能。

全屏动态视觉层不放在右侧，而是作为整个首屏的空间氛围。它包含：

- 微弱网格。
- 细线轨迹。
- 少量坐标点。
- 慢速扫描光。
- 低透明度漂浮关键词，如 `AISDLC`、`Agent`、`MCP`、`Frontend`、`Review`。

这些元素是视觉层，不承担主要阅读任务，不能抢过姓名和身份。

底部事实刻度层用于补足可信度，替代原来的详细信息卡。它以横向刻度或低调数据栏呈现 3-4 个事实：

- `6 年前端`
- `美团 / 小米`
- `AISDLC`
- `性能优化 50%`

事实刻度需要像封面的一部分，而不是卡片列表。

联系方式保留在页面底部联系区，不出现在首屏。

## 动效设计

进入页面时按视觉层级播放：

1. 背景网格和轨迹先低透明度出现。
2. 姓名主标题进入，作为最强节奏点。
3. 身份、摘要和按钮随后 stagger 进入。
4. 漂浮关键词、坐标点和事实刻度轻量出现。

桌面端允许：

- 鼠标移动时，轨迹、关键词和坐标点做轻微视差。
- 扫描光慢速掠过首屏。
- 背景线条做非常慢的漂浮或位移。
- 按钮保留高光扫过和轻微位移反馈。
- 滚动离开首屏时，背景层可有轻微反向视差。

移动端需要：

- 减少漂浮关键词和坐标点数量。
- 降低位移幅度。
- 不依赖鼠标交互。
- 保证标题、摘要、按钮和事实刻度不重叠。

`prefers-reduced-motion: reduce` 下：

- 关闭连续循环动画。
- 关闭鼠标视差。
- 关闭大幅位移和扫描光。
- 保留近似静态的首屏布局和必要的短入场。

## 技术设计

继续使用现有 `GSAP + ScrollTrigger`。不新增动画依赖。

React 层面：

- 重组 Hero DOM。
- 移除当前首屏右侧候选人详细信息卡。
- 新增中心标题组、全屏背景视觉层、底部事实刻度。
- 使用现有 `useRef` 和 `useEffect` 管理 Hero 动画。
- 使用 `gsap.context()` 限定动画作用域。
- 清理鼠标监听、timeline 和 ScrollTrigger，避免热更新后重复绑定。

CSS 层面：

- 将 Hero 改为无左右栏布局。
- 背景视觉层铺满首屏，并通过 `z-index` 位于文字之后。
- 主标题和按钮始终是可读的前景层。
- 使用稳定尺寸、响应式约束和 `overflow` 控制，避免动态元素造成横向溢出。
- 只对需要动画的视觉元素添加 `will-change`。
- 强化 `prefers-reduced-motion` 样式。

## 验收标准

1. 首屏不再是左右布局。
2. 首屏不展示电话、邮箱等详细个人信息。
3. 第一眼能看出未来感个人品牌和 AI 前端方向。
4. 动画比当前更丰富，但不呈现工作台、Dashboard 或信息卡堆叠感。
5. 姓名、身份、摘要和按钮是首屏最清晰的阅读路径。
6. 底部事实刻度能补足可信度，并自然引导继续滚动。
7. 移动端没有横向溢出，文字和按钮不重叠。
8. `prefers-reduced-motion: reduce` 下没有持续运动和明显视差。
9. `npm run build` 通过。
